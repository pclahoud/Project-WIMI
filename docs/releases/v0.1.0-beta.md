# WIMI v0.1.0-beta — First Public Release

**WIMI (What I Missed It)** is a metacognitive exam preparation tool for high-stakes standardized exams. Instead of just tracking *that* you missed a question, WIMI helps you analyze *why* — and weighs your weaknesses against how heavily each topic actually counts on your exam.

This is the first public release after several months of private development. WIMI is stable for personal daily use; it's marked beta because it has so far had an audience of one.

## Highlights

**Structured mistake analysis**

- Session-based entry workflow with rich text (tables, KaTeX math), media attachments, auto-save, and fuzzy subject search
- Each entry captures what you picked, the correct answer, your reasoning, and the *kind* of mistake (knowledge gap, misread stem, time pressure, second-guessing)
- Multiple assignable notes per entry, linkable to specific subjects

**Exam modeling that matches reality**

- Multi-dimensional categorization — tag one question along several axes at once (organ system × physician task × discipline, USMLE-style)
- Subject hierarchies are true DAGs: shared subjects can appear under multiple parents with per-edge weights
- Hamilton largest-remainder question allocation, feasibility checking, and support for computer-adaptive (variable-length) exams
- 10 built-in templates: USMLE Step 1/2 CK, NBME shelf exams, MCAT, SAT, ACT, GRE, LSAT, CPA — fully customizable

**Analytics**

- D3.js sunburst, cross-dimension heatmaps, activity trends, and streak tracking
- Weakness-vs-exam-weight quadrant analysis and pattern detection with study recommendations
- Per-subject deep-dive pages with parent-context disambiguation

**Platform**

- Local-first: everything lives in SQLite on your machine — no account, no cloud, no telemetry
- Windows and macOS; run from source (`pip install -r requirements-prod.txt && python run_wimi.py`) or build a standalone executable with the included PyInstaller scripts
- Plugin system with a permission-scoped API (Python backend plugins, JS/CSS frontend plugins)
- Versioned database migrations — your data survives upgrades

## Known issues

Beta means honest: a few entry-form rough edges are known and tracked — occasional auto-scroll while typing in long entries, section headers sometimes needing a second click, and table right-click menus inside the rich text editor being unreliable. See the issue tracker for current status.

## Feedback

Bug reports and feature requests are welcome via GitHub Issues. If you're using WIMI for an exam not covered by the built-in templates, we'd especially like to hear how the hierarchy modeling holds up.
